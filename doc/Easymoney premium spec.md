## 1. 課金モデル

| 項目 | 内容 |
|------|------|
| 初期リリース | 買い切り（In-App Purchase） |
| 価格帯 | ¥1,000 〜 ¥1,500 |
| 将来追加 | 月額サブスク ¥290〜300 |
| 無料トライアル | なし（オンボーディングで訴求） |

初期は買い切りでリリースし、将来的にサーバーコストを伴う新機能（AI連携・クラウド同期等）を追加する場合にサブスクプランを別途追加する。

## 2. Free / Premium 機能比較

### 2.1 制限付き機能（Free → Premium で拡張）

| 機能 | Free | Premium |
|------|------|---------|
| 支出カテゴリ | 7個 | 最大10個 |
| 収入カテゴリ | 2個（固定収入・臨時収入） | 最大10個 |
| カスタムコイン | 2個 | 最大11個 |

コイン表示枚数はユーザーが設定画面のhiddenCoinsで管理する。デフォルトコインを全て非表示にしてカスタムコイン11個のみの構成も可能。

### 2.2 Premium 限定機能（8個）

| 機能 | 種別 | 説明 |
|------|------|------|
| カテゴリ拡張 | 課金ゲート | 支出最大10個 / 収入最大10個まで追加可能 |
| カスタムコイン拡張 | 課金ゲート | 最大11個まで追加可能 |
| テーマカラー変更 | 課金ゲート | 複数のカラーテーマから選択 |
| CSVエクスポート | 新機能 | 履歴データをCSVファイルに書き出し |
| グラフ詳細 | 新機能 | カテゴリ別推移・月次比較グラフ |
| 定期取引の自動入力 | 新機能 | 家賃・サブスク等を毎月自動記録 |
| カテゴリ別予算設定 | 新機能 | 食費3万・交通費1万…のように個別管理 |
| 広告非表示 | 将来検討 | Freeプランに広告を導入した場合 |

### 2.3 無料で提供する機能

| 機能 | 説明 |
|------|------|
| ウィジェットカスタマイズ | 表示内容・レイアウトの変更 |
| 位置情報の自動記録 | 取引に場所を自動付与 |
| 年間レポート | 1年の振り返りをビジュアルで生成 |

## 3. 課金UXフロー

課金導線は「A. オンボーディング」と「B. ペイウォール」の2つに明確に分離する。

### 3.1 A. オンボーディング（初回起動時のみ）

| 項目 | 内容 |
|------|------|
| 表示タイミング | アプリ初回起動時に1回だけ |
| スライド数 | 20枚以上（5フェーズ構成） |
| スキップ | スライド間のスキップ不可（順番に見る） |
| 離脱 | ×ボタンでいつでも離脱可能 → ホーム画面へ |
| 購入ボタン | 全スライドで画面下部に常時表示 |
| 購入時 | 祝福アニメーション → ホーム画面 |
| 再表示 | 離脱した場合、次回起動で再表示しない（1回きり） |
| パーソナライズ | Phase 2の回答をPhase 3〜5で再利用 |

### 3.2 B. ペイウォール（Pro機能タップ時）

| 項目 | 内容 |
|------|------|
| 表示タイミング | Proマーク付き機能をタップした時（毎回） |
| 表示内容 | タップした機能のプレビュー + 購入ボタン（1枚） |
| Proマーク | UI上でプレミアム機能と識別できるバッジ |
| 離脱 | 「閉じる」で元の画面に戻る |
| 購入時 | 祝福アニメーション → タップした機能の画面に遷移 |
| トリガー箇所 | 全8箇所（カテゴリ追加、コイン追加、テーマ変更、CSVエクスポート、グラフ詳細、定期取引、カテゴリ別予算、広告非表示） |

## 4. オンボーディング スライド詳細

海外の成功事例（Life Reset, Prayer Lock, QUITTR）を参考に、パーソナル質問→課題認識→機能紹介→クロージングの流れで構成。購入ボタンは全スライドで常時表示。

### Phase 1: フック — 感情をつかむ（3枚）

最初の数枚でユーザーの気持ちを掴む。EasyMoneyの世界観を伝えつつ「自分ごと」にさせる。

**スライド 1: 「お金のこと、ちゃんと見えていますか？」** [問いかけ]
> 大きな文字で問いかけ。アプリのメイン画面（ラジアルメニュー）がうっすら背景に。考えさせる導入。

**スライド 2: 「家計簿をつける人は平均¥X節約」** [統計データ]
> 一般統計データを引用して数字をドーンと表示（総務省家計調査等）。アニメーションで数字がカウントアップ。社会的証明。

**スライド 3: 「3秒で記録。だから続く。」** [価値提示]
> コインタップのアニメーション。EasyMoneyのコア体験を再確認。「ここまでは無料で使えてます」の安心感。

### Phase 2: パーソナル質問 — コミットさせる（4枚）

ユーザーに自分の状況を入力させて心理的コミットメントを高める。回答内容は後のスライドでパーソナライズに使用。

**スライド 4: 「毎月の支出、把握できていますか？」** [質問（選択式）]
> 「だいたい把握」「あまり把握していない」「全然わからない」の3択。ほとんどの人が「あまり」を選ぶ → 課題認識へ。

**スライド 5: 「一番お金を使いすぎてると思うのは？」** [質問（カテゴリ選択）]
> 食費・交際費・日用品・交通費・その他をタップ。ラジアルメニュー風UIで選ばせる（アプリの操作感を先取り体験）。

**スライド 6: 「毎月いくら節約したいですか？」** [質問（金額スライダー）]
> スライダーで¥5,000〜¥50,000を選択。入力した金額を後のスライドで「あなたの目標: ¥XX,XXX」と表示。

**スライド 7: 「家計管理で一番ストレスなのは？」** [質問（選択式）]
> 「記録が面倒」「何に使ったか忘れる」「予算オーバーに気づけない」「カテゴリが合わない」。ペインポイントを自覚させる。

### Phase 3: 課題認識 — 痛みを可視化（3枚）

Phase 2の回答を使って「あなたの現状」を見せる。統計データと組み合わせて危機感を高める。

**スライド 8: 「把握していない支出は意外と大きい」** [パーソナライズ結果]
> 一般統計ベースで「家計を把握していない世帯は月平均¥X,XXXの無駄遣い」+ Q6の目標額を再表示「あなたの目標: 月¥XX,XXXの節約」。

**スライド 9: 「カテゴリが足りないと "その他" に埋もれる」** [課題提示]
> 円グラフで「その他 42%」という状態を見せる。「本当は医療費・趣味・美容…細かく分けたいのに」。Freeの限界を自然に認識させる。

**スライド 10: 「同じ金額、毎月手入力してませんか？」** [課題提示]
> 家賃・サブスク・光熱費…繰り返し入力するストレスを描写。「自動で記録できたら？」と次フェーズへ橋渡し。

### Phase 4: 解決策 — Premium の価値を見せる（7枚）

課題に対する具体的な解決策を、アプリのモックアップ付きで見せる。各スライド1機能。

**スライド 11: 「カテゴリを自由に追加。"その他" をなくそう」** [機能紹介 — カテゴリ拡張]
> 10個のカテゴリが並ぶラジアルメニューのモック。Q5で選んだカテゴリがハイライト。「あなたの○○費も、専用カテゴリで管理」

**スライド 12: 「よく使う金額を、コインにしよう」** [機能紹介 — コイン拡張]
> カスタムコインが並ぶCoinListのモック。「ランチ ¥850」「コンビニ ¥300」のような実用例。ワンタップ記録の体験を想像させる。

**スライド 13: 「毎月の固定費、もう手入力しなくていい」** [機能紹介 — 定期取引]
> 定期取引リストのモック（家賃、Netflix、スマホ代…）。設定UIイメージ。「一度設定すれば毎月自動」

**スライド 14: 「食費3万、交際費1万。カテゴリごとに予算管理」** [機能紹介 — カテゴリ別予算]
> カテゴリ別プログレスバーのモック。「食費: ¥24,800 / ¥30,000」。Q5で選んだカテゴリでパーソナライズ。

**スライド 15: 「お金の流れが、もっと見える」** [機能紹介 — グラフ詳細]
> カテゴリ別推移の折れ線グラフモック。月ごとに食費が下がっていくアニメーション。「推移も比較も」

**スライド 16: 「データはあなたのもの。CSVで書き出し」** [機能紹介 — CSVエクスポート]
> CSVファイルアイコン + Excelで開いたイメージ。確定申告・家計見直しに活用。安心感。

**スライド 17: 「見た目も、あなたらしく」** [機能紹介 — テーマカラー]
> 複数テーマの並列プレビュー。同じラジアルメニューが違うカラーで表示される。ビジュアルインパクト大。

### Phase 5: クロージング — 背中を押す（3枚）

全体をまとめて購入への最後のひと押し。

**スライド 18: 「Premiumでできること」まとめ** [機能まとめ]
> 全Premium機能のチェックリスト形式。レビューが溜まったらユーザーの声（★5レビューカード）に差し替え。

**スライド 19: 「あなたにぴったりの使い方」** [パーソナライズまとめ]
> Q4-7の回答を基に「あなたが特に活用できる機能」をハイライト表示。目標節約額を再表示。

**スライド 20: 「Premiumをはじめる」** [購入CTA画面]
> 買い切り価格をドーンと表示。「¥1,280で、ずっと使える」。月額換算「1日たった¥3.5」。購入ボタン大きく。

## 5. オンボーディング設計原則

| 原則 | 詳細 |
|------|------|
| 購入ボタン常時表示 | 画面下部に固定配置。どのスライドからでも即購入可能。 |
| スキップ不可 | スライド間の飛ばし不可（順番に見る必要がある）。全体の離脱は×ボタンで可能。 |
| プログレスバー | 上部に進捗を表示。残りスライド数が分かる安心感。 |
| パーソナライズ | Phase 2の回答をPhase 3-5で再利用。「あなたの○○」と動的表示。 |
| アニメーション | 各スライドにreanimated v4で微アニメーション。数字カウントアップ、フェードイン等。 |
| デザイン | アプリ本体と同じPalette使用。実際のUI要素のモックを積極的に見せる。 |
| 離脱後の復帰 | ×で離脱 → 次回起動では表示しない（1回きり）。設定画面からオンボーディングを再表示する導線は用意する。 |
| 統計データ | リリース前は一般統計データのみ使用。架空のユーザー実績は掲載しない。 |

## 6. ペイウォール（Pro機能タップ時）設計

### 6.1 表示内容

1枚のモーダルダイアログで構成。タップした機能ごとにプレビュー画像を動的に切り替える。

| 要素 | 詳細 |
|------|------|
| 機能名 | タップした機能の名称（例:「CSVエクスポート」） |
| プレビュー画像 | 機能のモックアップ or スクリーンショット（機能ごとに異なる） |
| 説明テキスト | 「この機能はPremiumで利用できます」+ 機能の1行説明 |
| 購入ボタン | 買い切り価格を明記（例:「¥1,280でアップグレード」） |
| 閉じるボタン | 「閉じる」で元の画面に戻る |

### 6.2 Proマーク

Premium機能のUI要素には事前にProマーク（バッジ）を表示し、タップ前にPremium機能だと識別できるようにする。これにより誤タップによるイライラを防止する。

### 6.3 トリガー箇所（全8箇所）

| トリガー | 発生画面 | 条件 |
|----------|----------|------|
| カテゴリ追加（支出） | 設定 → カテゴリ管理 | 8個目の支出カテゴリ追加時 |
| カテゴリ追加（収入） | 設定 → カテゴリ管理 | 3個目の収入カテゴリ追加時 |
| カスタムコイン追加 | 設定 → コイン設定 | 3個目のカスタムコイン追加時 |
| テーマカラー変更 | 設定 → テーマ | テーマ変更をタップ時 |
| CSVエクスポート | 履歴画面 | エクスポートボタンタップ時 |
| グラフ詳細 | 履歴画面 | 詳細グラフ表示をタップ時 |
| 定期取引設定 | 設定 → 定期取引 | 設定画面をタップ時 |
| カテゴリ別予算 | 設定 → 予算管理 | カテゴリ別予算設定をタップ時 |

## 7. 技術実装方針

### 7.1 課金基盤

| 項目 | 方針 |
|------|------|
| 決済 | Apple In-App Purchase（Non-Consumable = 買い切り） |
| ライブラリ | expo-in-app-purchases or react-native-iap |
| 購入状態の保存 | Firestore settings/user_settings_v1 に isPremium: boolean を追加 |
| レシート検証 | サーバーサイド検証推奨（Firebase Functions等） |
| リストア | 「購入を復元」ボタンを設定画面に配置 |

### 7.2 新規ファイル（予定）

| ファイルパス | 役割 |
|-------------|------|
| src/services/purchaseService.ts | 課金処理（購入・復元・状態確認） |
| src/hooks/usePremium.ts | Premium状態を管理するカスタムフック |
| src/screens/Onboarding/ | オンボーディング画面一式 |
| src/screens/Onboarding/hooks/useOnboarding.ts | スライド制御・質問回答の管理 |
| src/screens/Onboarding/components/ | 各スライドのコンポーネント |
| src/components/PaywallModal.tsx | ペイウォールモーダル（共通） |
| src/components/ProBadge.tsx | Proマークバッジ（共通） |
| src/constants/premium.ts | Premium機能の定義・制限値 |

### 7.3 既存ファイルへの変更

| ファイルパス | 変更内容 |
|-------------|----------|
| src/index.ts | PremiumFeature型、OnboardingAnswer型を追加 |
| src/services/masterService.ts | カテゴリ・コイン追加時のPremium制限チェック |
| src/constants/categories.ts | DEFAULT_CATEGORIESを7個に拡張（Free上限） |
| app/_layout.tsx | 初回起動判定 → オンボーディング画面へルーティング |
| src/screens/Settings/ | 各設定項目にProマーク・ペイウォール導線を追加 |
| src/screens/MoneyHistory/ | CSVエクスポート・グラフ詳細にProマーク追加 |

### 7.4 Firestoreデータ構造の拡張

settings/user_settings_v1 ドキュメントに以下のフィールドを追加:

| フィールド | 型 | 説明 |
|-----------|-----|------|
| isPremium | boolean | Premium購入済みかどうか（デフォルト: false） |
| purchasedAt | Timestamp \| null | 購入日時 |
| onboardingCompleted | boolean | オンボーディング完了済みかどうか |
| onboardingAnswers | object \| null | Phase 2の回答データ（パーソナライズ用） |

既存のFirestoreデータ構造（households, items, settings）は変更しない。
